Stage 1: The Basics of the Web and HTML

What is the World Wide Web (www) made up of?

  • Invented in the 1990's
  • Created with HTMLcoding
    • HTML = HyperTextMarkupLanguage
  • Hyperlinks which glue the documents together
  • Consists of over 30 billion pages

What are tags and Elements

  • Open Tags are contained within brackets
  • Closing Tags are duplicates of the opening tag, but preceded by a forward slash
  • An element is the entire opening tag, contents and closing tag
  • Attributes are content properties of HTML elements

Why are computers stupid?

  • Computers only think via linear concepts
  • If all lines are coded correctly, but the syntax is wrong errors and debugging will follow

HTML Tag Library

  • Inline Tags
    • bold
    • emphasis
    • image
  • Block Tags
    • ordered list
    • paragraph
    • unordered list

Stage 2: Creating a Structured Document with HTML

Front End Web Development

How to translate a visual design into a website

  • HTML code represents the structure of our website
  • CSS files represent the style our website displays
  • JavaScript influences the interactive parts of the website

Tree-like structure of a webpage

  • HTML elements may be nested within one another to form layers or branches
  • This tree-like structure is also know as the DOM or Document Object Model

Boxes, Boxes and more Boxes

  • The div tag is used to group block-elements to be formatted with CSS stylings
    • I picture div as both the top and the bottom of the box with CSS as the depth and width of the box

Programmers use various code editing software programs

  • Scratchpad
  • CodePen
  • Sublime 2.0.2
    • We will use Sublime until further notice

Stage 3: Adding CSS Style to HTML Structure

  • Style is added by using CSS coding
    • CSS = Cascading Style Sheets
  • CSS allows a programmer to avoid repetition
    • Repetition is avoided by giving similar HTML elements the same “class”, by defining the “styling” for the class with CSS; we do so only the one time, the style gets propagated to any instance of the class element so named regardless of how many time it is called.
    • Regarless of how mnay times we change the single line of CSS code it will always change every instance of the HTML code so named.
  • CSS acts as a search and replace tool
  • Order Matters in the definition of rules and in what order they are applied
  • Specifics matter such that cascading rules apply the most specific rule first

Random Takeaways

Five ways to think like a programmer

  1. Procedural Thinking
    • Big Problem turned into logical steps
  2. Abstract Thinking
    • Symbols may have numberical values
    • Numbers may have symbolic meaning
  3. Systematic Debugging
    • Identify the problem
    • Trackdown possible causes
  4. Technological Empathy
    • Understanding how computers send things
  5. Systems Thinking
    • One big item broken into component parts

Debugging Methodology

  • Collect evidence
  • Generate theories
  • Test those theories
  • Fix the problem
  • Rinse and repeat

Programmers have the greatest help desk available to them in the Internet

  • Humans are likely to forget some of the things they once learned
  • To learn to use previously unknown Tags one need only query the Internet
    • I learned to use ordered and unordered lists for this exercise

4 Stages of Expertise

  1. Ignorance - know nothing
  2. Awareness - matter of minutes to know it exists
  3. Ability - days to begin doing it but still mind specific
  4. Fluency - typically years of practice are required
    • Not thinking only doing

During the course we will not become fluent, but will strive to reach the Ability level

When we hit the wall - how may we stick with it?

  1. Ask for help
    • Check with Google
    • Use the discussion forums
  2. Take a break
  3. Backchain the answer
    • Learn the answer first then take all of the preceding steps necessary to reach it
